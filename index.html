O que é o Plante Árvores
🌱

Doação no carrinho

Checkbox opcional no minicart e no checkout legado. O cliente decide se deseja contribuir com um valor fixo por pedido.

⚙️

Provisionamento automático

O app cria toda a estrutura de produto via APIs VTEX — categoria, marca, SKU, preço e estoque. Nenhuma ação manual no catálogo.

📊

Dashboard de doações

Painel no admin com histórico de pedidos que incluíram doação, valores arrecadados e status em tempo real.

Instalação na loja
1
Instalar o app via VTEX CLI
CLI
terminal
vtex install performaitpartnerbr.donation-app@1.0.0
ℹ️ O app requer os builders node 7.x, react 3.x, store 0.x e admin 0.x no manifest.json do tema.
2
Adicionar ao manifest do store-theme
manifest.json

Adicione o app como peerDependency no manifest.json do seu store-theme:

store-theme/manifest.json
"peerDependencies": {
  "performaitpartnerbr.donation-app": "1.x"
}
3
Adicionar o bloco no template do minicart
blocks.jsonc

Adicione o bloco donation-minicart dentro do template do minicart em store/blocks/minicart.jsonc:

store/blocks/minicart.jsonc
"flex-layout.row#donation": {
  "children": [
    "performaitpartnerbr.donation-app:donation-minicart"
  ]
}
⚠️ O bloco só é renderizado quando appStatus === 'ACTIVE'. Antes de ativar o app, nada é exibido no minicart.
3
Configuração no admin

Após instalar o app, acesse Admin → Doações → Cadastrar Produto e siga o fluxo abaixo:

1. Ver onboarding
→
2. Selecionar valor
→
3. Criar produto
→
4. Ativar o app
✅ O app cria automaticamente categoria, marca, produto, SKU, imagem, preço e estoque via APIs VTEX. Nenhuma ação manual no catálogo é necessária.
⚠️ O valor da doação não pode ser alterado após a criação do produto nesta versão. Escolha com atenção antes de confirmar.
ℹ️ Para lojas marketplace: ative a opção Notify Subaccounts em /admin/apps/vtex.orders-broadcast/setup para capturar pedidos de subcontas.
4
Instalação no checkout

O checkout legado da VTEX requer a instalação manual do script CDN. Escolha uma das opções:

A
Via Google Tag Manager (recomendado)
GTM

Crie uma tag do tipo Custom HTML com gatilho na página de checkout (/checkout):

GTM → Nova tag → Custom HTML
<script
  src="https://jefersonduartepit.github.io/donation-checkout-cdn/checkout-cdn/donation-checkout.js"
  data-sku-id="SEU_SKU_ID">
</script>
ℹ️ Substitua SEU_SKU_ID pelo ID do SKU gerado após o provisionamento. Encontre em Admin → Doações → Ver SKU no catálogo.
B
Via checkout6-custom.js
Portal Admin

Acesse /admin/portal/#/sites/default/code/files/checkout6-custom.js e adicione ao final do arquivo:

checkout6-custom.js
// === Plante Árvores — Donation App ===
var s = document.createElement('script')
s.src = 'https://jefersonduartepit.github.io/donation-checkout-cdn/checkout-cdn/donation-checkout.js'
s.setAttribute('data-sku-id', 'SEU_SKU_ID')
document.head.appendChild(s)
// === fim Donation App ===
5
Verificação

Confirme que tudo está funcionando corretamente:

  • Admin exibe "Produto criado com sucesso!" após o provisionamento
  • Toggle de ativação habilitado e responsivo
  • Checkbox de doação aparece no minicart do storefront
  • Checkbox aparece antes do botão "Finalizar pedido" no checkout
  • Estado do checkbox sincronizado com o orderForm ao recarregar
  • Pedido com doação aparece no Dashboard de Vendas após finalização
  • Para lojas marketplace: "Notify Subaccounts" ativado no orders-broadcast
